## 前提条件

- GitHubアカウント
- 対象リポジトリへの適切なアクセス権限

## セットアップ手順

### Kommonへのログイン

1. [Kommon管理ページ](https://kommon-admin.vercel.app/)にアクセスします。
2. 「GitHub で認証する」ボタンをクリックします。
3. GitHubの認証画面が表示されますので、指示に従って認証を完了してください。

![Kommonログイン画面のイメージ](https://example.com/images/kommon-login.png)

### 組織へのアクセス許可

1. ログイン後、Kommonを利用したい組織（Organization）または個人アカウントを選択します。
2. 「Grant」ボタンをクリックして、Kommonがその組織にアクセスするための許可を与えます。

![組織へのアクセス許可画面のイメージ](https://example.com/images/org-access.png)

### API情報の設定

1. 組織のダッシュボード画面で、「API設定」または同様のセクションに移動します。
2. 必要なAPI情報を入力します。これには以下が含まれる場合があります：
   - API キー
   - エンドポイント情報
   - その他の認証情報
3. 「保存」または「適用」ボタンをクリックして設定を完了します。

### GitHub Appのインストール

1. [Kommon AI GitHub App](https://github.com/apps/kommon-ai)ページにアクセスします。
2. 「インストール」または「Configure」ボタンをクリックします。
3. Kommonを使用したい組織またはユーザーアカウントを選択します。
4. アクセス設定画面で、Kommonを利用したいリポジトリを選択します：
   - すべてのリポジトリにアクセス許可を与える場合は「All repositories」を選択
   - 特定のリポジトリのみアクセス許可を与える場合は「Only select repositories」を選択し、対象リポジトリをリストから選択
5. 「Install」または「Save」ボタンをクリックして設定を完了します。

![GitHub Appのインストール画面のイメージ](https://example.com/images/github-app-install.png)

## Kommonの使い方

### 基本的な使い方

1. Kommonが設定されたリポジトリのissueを開きます。
2. Issue内にコメントとして「/kommon」で始まる指示を記入します。例：
   ```
   /kommon このコードの問題を修正して
   ```
   または
   ```
   /kommon 新しい機能を実装してください
   ```
3. コメントを投稿すると、Kommon AIがタスクを受け取り、処理を開始します。
4. KommonはGitHubに直接プッシュしたり、Pull Requestを作成したりする能力があります。

### リアクションの意味

Kommonとの対話では、以下のリアクションが使用されます：

- 👀 (eyes) - Kommonが指示を受け取り、作業を開始したことを示します
- 🚀 (rocket) - Kommonが指示された作業を完了したことを示します

## トラブルシューティング

もしKommonが期待通りに動作しない場合は、以下の方法をお試しください：

1. **基本的な確認**:
   - GitHubアプリが正しくインストールされているか確認する
   - 対象リポジトリへの適切な権限が付与されているか確認する

2. **動作テスト**:
   - 簡単なテストコマンドで動作確認する
   ```
   /kommon こんにちは
   ```

3. **手動での操作確認**:
   - GitHub APIの動作確認のため、小さな変更をプッシュしてみる
   - 小さなPull Requestを作成してみる

問題が解決しない場合は、[Kommonサポート](mailto:support@example.com)にお問い合わせください。
